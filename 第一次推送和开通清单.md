# 第一次推送和开通清单

## 先在本地执行

如果你当前网络能连 GitHub，在这个目录里执行：

```powershell
git push -u origin main
```

如果提示需要登录，就按 GitHub 的浏览器登录流程完成。

## GitHub 里要点的地方

推上去以后，到仓库：

- `Settings`
- `Pages`
- `Build and deployment`
- `Source` 选 `GitHub Actions`

然后到：

- `Actions`

确认 `Deploy Blog` 这个工作流开始运行。

## CloudBase 里要点的地方

1. 创建一个 CloudBase 环境
2. 开通静态网站托管
3. 记下：
   - `环境 ID`
   - `SecretId`
   - `SecretKey`

## GitHub 里要填的 Secrets

到仓库：

- `Settings`
- `Secrets and variables`
- `Actions`

新增 3 个仓库 Secrets：

- `TCB_SECRET_ID`
- `TCB_SECRET_KEY`
- `TCB_ENV_ID`

填完后，再随便改一篇文章、提交、push 一次，CloudBase 部署就会自动启用。

## 成功后你会有两个地址

GitHub Pages：

```text
https://beibeihk.github.io/myblog/
```

CloudBase：

```text
https://你的默认域名/myblog/
```

## 以后更新博客

以后就是这套：

```powershell
git add .
git commit -m "update blog"
git push
```

GitHub Pages 和 CloudBase 会一起更新。
