# 中国大陆可访问部署指南

## 先说结论

你这个博客本质上是一个 `Hexo` 静态站，真正影响中国大陆访问的核心，不是 Hexo 本身，而是你现在的托管位置。

你当前项目里能看到：

- 站点地址现在写的是 `pages.dev`：`_config.yml`
- 部署方式还是推到 GitHub：`_config.yml`
- 主题里的大部分静态资源已经是国内友好源：`themes/fluid/_config.yml`

所以这件事的重点不是“改博客代码”，而是“把生成好的静态文件放到中国能稳定访问的地方”。

## 你现在为什么会这样

你当前配置更像是这一条链路：

1. 本地写 Hexo
2. 推到 GitHub
3. 由 Cloudflare Pages 或类似海外平台托管
4. 中国大陆用户访问 `pages.dev`

这类海外平台在中国大陆经常会出现下面这些情况：

- 能打开，但很慢
- 有些网络能开，有些网络打不开
- 图片、CSS、JS 偶发加载失败
- 完全需要 VPN

## 真正可行的两条路

### 方案 A：想要“最稳”，走中国大陆节点

这是最推荐的方案。

你需要：

- 自己的域名
- 中国大陆的静态托管或服务器
- ICP 备案
- 最好再接国内 CDN

适合：

- 你希望中国大陆用户大多数都能稳定打开
- 你以后想长期写博客
- 你愿意花一点时间做备案

常见做法：

- 阿里云 OSS 静态网站托管 + CDN
- 腾讯云 COS 静态网站托管 + CDN
- 轻量云服务器 + Nginx

### 方案 B：想要“先能直接打开”，走香港节点

这是最快落地的方案。

你需要：

- 自己的域名
- 香港区域的对象存储或服务器

特点：

- 一般不需要 ICP 备案
- 中国大陆大多数用户通常可以直接访问
- 但速度和稳定性通常不如“大陆节点 + 备案 + CDN”

适合：

- 你想先尽快上线
- 暂时不想做备案

常见做法：

- 阿里云香港 OSS/服务器
- 腾讯云香港 COS/服务器

## 如果你真的想做到“几乎所有人不用 VPN 都能看”

最接近这个目标的路线只有一条：

1. 买域名
2. 备案
3. 把 Hexo 生成后的 `public` 目录部署到中国大陆托管
4. 绑定自己的域名
5. 接国内 CDN

没有备案也能做，但做不到最稳。

## 你的 Hexo 项目需要注意的配置

### 1. `url` 要改成你最终正式域名

你现在 `_config.yml` 里写的是：

```yml
url: https://myblog-1k8.pages.dev/myblog
```

如果你以后用自己的独立域名，比如：

```yml
url: https://blog.example.com
root: /
```

如果你未来真的部署在子路径，比如：

```yml
url: https://example.com
root: /myblog/
```

不要再把 `pages.dev` 当成长期正式地址。

### 2. 不要把根路径和子路径混着配

你现在这行：

```yml
url: https://myblog-1k8.pages.dev/myblog
```

只有在“站点真的部署在 `/myblog/` 子目录”时才合理。  
如果以后你换成独立域名根目录部署，通常应该改成：

```yml
url: https://你的域名
root: /
```

### 3. `deploy` 可以不再依赖 GitHub Pages 这条链路

你现在是：

```yml
deploy:
  type: git
  repo: git@github.com:beibeihk/myblog.git
  branch: main
```

这说明你当前部署思路还是围绕 GitHub。  
如果改成阿里云 OSS / 腾讯云 COS，这段配置可以先保留不用，后面再换成新的自动化发布方式。

## 最实际的落地步骤

### 路线 1：最快能用

1. 买一个域名
2. 选一个香港节点托管
3. 在本地生成静态文件
4. 把 `public` 目录上传
5. 绑定域名
6. 开 HTTPS
7. 测试中国大陆手机网络和家宽访问

本地生成命令：

```powershell
npm run clean
npm run build
```

要上传的内容：

```text
blog/public/
```

### 路线 2：最稳长期方案

1. 买域名
2. 选择阿里云或腾讯云大陆节点
3. 做 ICP 备案
4. 开通静态网站托管
5. 上传 `public` 目录
6. 绑定域名
7. 接 CDN
8. 测试全国访问

## 你的项目目前反而有利的一点

你这个主题里的很多前端资源已经不是走 Google、jsDelivr 这类国外源，而是国内可访问的源，例如：

- `lib.baomitu.com`
- `at.alicdn.com`
- `registry.npmmirror.com`

这说明你迁移到国内托管后，前端资源这块阻力不大。

## 你现在最值得优先做的事

如果你想少走弯路，优先级建议是：

1. 先决定是“香港快速上线”还是“大陆备案长期稳定”
2. 申请一个你自己的域名
3. 把 `_config.yml` 里的正式地址改成未来要用的域名
4. 用 `npm run build` 生成静态文件
5. 把 `public` 上传到新托管平台
